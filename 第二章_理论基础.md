# 第2章 相关理论与工具基础

本章把后面用到的理论一次性讲清楚，不绕弯。每一节先说这个东西要解决什么问题，再给公式，最后说本文里怎么用。

## 2.1 推荐系统的形式化与隐式反馈

推荐问题可以这样写：有用户集合 U = {u₁, …, u_M}，有物品集合 I = {i₁, …, i_N}，二者之间的历史交互记成矩阵 R ∈ ℝ^{M×N}。要做的事情是学一个函数 f: U × I → ℝ，对任意没交互过的(u, i)给一个分数，按分数从高到低排序就得到推荐列表。

交互信号有两种。一种是显式反馈，R_{u,i} 是用户给物品打的分数（比如一到五星）；另一种是隐式反馈，R_{u,i} 只取 0 或 1，记录有没有发生过交互。本文用隐式反馈。理由很直接：业务里看到点击远比看到评分多；用户也不会主动给每件商品打分；做排序的时候，"会不会点"比"打几分"更有用。

隐式反馈下用二元交叉熵作损失。设模型输出概率 ŷ_{u,i} = σ(f(u,i))，σ 是 sigmoid，那么单个样本的损失为公式 (2.1)：

L_BCE(u, i, y) = − [ y · log ŷ_{u,i} + (1 − y) · log(1 − ŷ_{u,i}) ]    (2.1)

y ∈ {0, 1} 是标签。正样本来自交互记录 R_{u,i} = 1，负样本要从用户没交互过的物品里随机采样一些出来凑齐。本文每个正样本配 12 个负样本。

评估时用留一法：每位用户的最后一次交互拿来当测试样本，倒数第二次当验证样本，其他全是训练样本。测试时每个测试样本配 99 个用户没交互过的物品当负样本，模型对这 100 个候选打分，按分数从高到低排序，看正样本排到了第几名。具体的 HR@K 与 NDCG@K 算法在 2.7 节给出。

## 2.2 神经协同过滤模型

协同过滤是推荐的老办法，思路是相似的用户喜欢相似的物品，相似的物品被相似的用户喜欢。早期用矩阵分解，把交互矩阵 R 分解成两个低维矩阵的乘积，每个用户有一个 D 维向量 e_u，每个物品有一个 D 维向量 e_i，预测分数就是它们的内积 e_u · e_i。这种做法能处理稀疏性，但只能拟合线性关系。

神经协同过滤（NCF）把内积换成了多层感知机，这样模型就能拟合非线性的交互关系。NCF 的预测公式见公式 (2.2)：

ŷ_{u,i} = σ( W_L · φ_L( … φ_1([e_u ⊕ e_i]) … ) )    (2.2)

⊕ 表示把两个向量首尾拼接起来。φ_l(x) = ReLU(W_l x + b_l) 是第 l 层非线性变换，W_L 是输出层权重，σ 是 sigmoid 把输出压到 0 到 1 之间。本文取嵌入维度 D = 32，多层感知机隐藏层从 96 经 32、16 递减到 8（96 = 32 × 3 是因为后面要做三段拼接）。

本文在 NCF 上做了两处改造。第一，联邦设定下每个客户端只有一个用户，所以传统 NCF 中 M × D 的用户嵌入矩阵被简化为单行向量 e_u ∈ ℝ^D。第二，引入短期兴趣向量 s_u（怎么算见 2.3），输入从两段 [e_u ⊕ e_i] 改为三段 [e_u ⊕ s_u ⊕ e_i]，多层感知机的输入维度从 2D 变成 3D。这样模型既能记住用户的长期偏好，也能对当下感兴趣的东西做出反应。

训练用 SGD 优化器，损失就是公式 (2.1) 的 BCE。本文针对推荐稀疏更新的特点用了三个独立优化器，多层感知机权重、用户嵌入、物品嵌入分别管，三个学习率不一样。具体设法在 3.4 节训练协议里讲。

## 2.3 注意力机制与多头注意力

注意力解决的是"在一堆候选信息里挑出当前最相关的那部分"。打个比方，要预测用户下一步会不会买一双跑鞋，那他历史上买跑鞋、跑步袜、运动短裤的记录就比买毛衣的记录更有参考价值。注意力机制能自动算出这种"参考价值"。

具体怎么算？给三组向量：查询 Q ∈ ℝ^{n_q × d_k}、键 K ∈ ℝ^{n_k × d_k}、值 V ∈ ℝ^{n_k × d_v}。先用查询和每个键做内积得到相关度分数，分数除以 √d_k 之后过 softmax 转成权重，最后用权重对值做加权求和。公式见 (2.3)：

Attention(Q, K, V) = softmax( Q K^⊤ / √d_k ) V    (2.3)

这里除以 √d_k 是为了防止维度大了之后内积值膨胀，softmax 进入饱和区导致梯度消失。这个除数叫缩放系数。

单头注意力只在一个子空间里挑相关。多头注意力的想法是：把查询、键、值分别投影到 H 个不同子空间，每个子空间独立做一次注意力，再把 H 个头的输出拼接起来过一个输出投影矩阵融合。这样不同的头可以关注不同方面，比如一个头关注品类相似性、另一个头关注价格区间。多头注意力的公式见 (2.4) 与 (2.5)：

head_h = Attention( Q W_h^Q, K W_h^K, V W_h^V )    (2.4)

MultiHead(Q, K, V) = W^O · Concat(head_1, head_2, …, head_H)    (2.5)

W_h^Q, W_h^K, W_h^V ∈ ℝ^{D × d_h} 是第 h 个头的三个投影矩阵，d_h = D / H 是每个头的维度，W^O ∈ ℝ^{D × D} 是最后融合用的输出投影矩阵。

本文怎么用？用户的本地历史交互序列对应的物品嵌入当键和值，当前候选物品的嵌入当查询，多头注意力算出来的输出就是短期兴趣向量 s_u。换句话说，模型在预测用户对一件新物品的偏好时，会回头看历史上哪些物品最像它，然后从这些相似历史物品的嵌入里抽取信息。

为了让模型知道历史交互的先后顺序，本文给序列加了可学习的位置编码 P ∈ ℝ^{L_max × D}。L_max 是位置编码矩阵能容纳的最大长度，本文取 20。位置编码加在键值上，不加在查询上——查询代表当前候选物品，本身不属于这个序列，没有时序信息。带位置编码的键见公式 (2.6)：

K' = K + P[1:L]    (2.6)

L 是当前实际历史长度，本文取 5。P 初始化用均值 0、标准差 0.02 的高斯分布，初始值小是为了不在训练前期对序列建模引入太大扰动。注意力头数取 H = 2，每头维度 d_h = 16。

## 2.4 图神经网络与消息传递

图神经网络的核心动作是消息传递：每个节点的新表征等于自己加上邻居加权平均后再过一层非线性。这样信息可以沿着图的边逐步扩散——一层只能看到一跳邻居，两层就能看到二跳邻居。最常见的图卷积更新规则见公式 (2.7)：

h_v^{(l+1)} = σ( ∑_{u ∈ N(v) ∪ {v}} (1 / c_{vu}) · W^{(l)} h_u^{(l)} )    (2.7)

h_v^{(l)} ∈ ℝ^d 是节点 v 在第 l 层的表征，N(v) 是 v 的邻居集合，c_{vu} 是归一化系数（保证度数不同的节点之间公平），W^{(l)} ∈ ℝ^{d × d} 是这一层的参数矩阵，σ 是非线性激活。

本文怎么用？服务端建一个用户关系图，节点就是参与本轮训练的用户，边权由双图筛选得出的可信邻居关系决定（详见第 3、4 章）。在这个图上做消息传递，但跳过了 W^{(l)} 这一通道线性变换——保留嵌入语义不变，只做加权平均。每位用户的物品嵌入张量更新规则见公式 (2.8)：

E_u^{(l+1)} = ∑_{v ∈ N_T(u) ∪ {u}} (A_{uv} / D_u) · E_v^{(l)}    (2.8)

N_T(u) 是 u 的可信邻居集合，A 是 0-1 邻接矩阵（A_{uv} = 1 当且仅当 v ∈ N_T(u)），D_u = ∑_v A_{uv} 是 u 的度数，E_v^{(l)} ∈ ℝ^{N × D} 是 v 在第 l 层的物品嵌入张量。这个更新可以写成矩阵形式 E^{(l+1)} = D^{−1} A E^{(l)}，等价于行归一化邻接矩阵右乘所有用户的物品嵌入张量。本文做 2 层消息传递，再多就会过平滑——所有用户的物品嵌入趋于一致，反而损害个性化。

为什么用行归一化（每行除以该行之和）而不用对称归一化？因为行归一化能保证每位用户从邻居处接收的信息量恒等于 1，避免高度数邻居的影响过分扩散到全局。

## 2.5 联邦学习

联邦学习的设计目标是数据不出本地、模型协同训练。最经典的联邦平均算法每一轮做四件事：服务端把当前全局模型 w^{(t)} 发给本轮采样到的客户端集合 P_t；每位客户端 k 在自己的本地数据 D_k 上做几轮 SGD，得到本地参数 w_k^{(t+1)}；客户端把 w_k^{(t+1)} 上传；服务端按各客户端的样本量做加权平均，公式见 (2.9)：

w^{(t+1)} = ∑_{k ∈ P_t} (n_k / n) w_k^{(t+1)}    (2.9)

n_k = |D_k| 是客户端 k 的样本数，n = ∑_k n_k 是所有参与客户端的总样本数。

联邦推荐的设定和典型联邦图像分类很不一样。第一，客户端粒度通常是单个用户，所以客户端数量从几十膨胀到几千、几万、几百万。第二，每位客户端的本地数据量极小，单个用户可能就几十到几百条交互。第三，物品集合在所有用户之间共享，所以物品嵌入需要在服务端聚合，但每位用户特有的用户嵌入和模型权重应该保持私有。

基于这些特点，本文采用个性化联邦推荐的范式：每位用户在本地维护私有的用户嵌入向量 e_u 和多层感知机权重 θ_u^{mlp}，每一轮训练从服务端下载上一轮的全局物品嵌入 E^{(t)}，在本地数据上做几轮 SGD 训练后，把训练后的物品嵌入上传到服务端聚合。这种设计在保护隐私的同时，让每位用户都有一个量身定做的模型。

## 2.6 差分隐私基础

数据不出本地不等于隐私自动得到保护。攻击者可以从客户端上传的物品嵌入或梯度里反向恢复出原始交互记录。要堵住这个漏洞，需要在算法层面加差分隐私。

### 2.6.1 (ε, δ)-差分隐私的形式定义

直观地说，差分隐私要求一个随机机制在两个只差一条记录的相邻数据集上输出几乎一样，从而保证从输出反推单条记录是否在数据集里这件事极其困难。形式上，对随机机制 M 与任意相邻数据集 D 与 D'，若对任意输出集合 S 满足公式 (2.10)：

Pr[M(D) ∈ S] ≤ e^ε · Pr[M(D') ∈ S] + δ    (2.10)

就说 M 满足 (ε, δ)-差分隐私。ε 叫隐私预算，越小越严；δ 是允许这个不等式被打破的小概率，通常取数据集大小倒数的量级。δ = 0 时简称 ε-DP，是更严格的形式。

### 2.6.2 Laplace 机制

实现差分隐私最简单的办法是给输出加噪声。给定一个数值型查询函数 f: D → ℝ^d，先算它的 L1 敏感度——任意两个相邻数据集上 f 输出向量的 L1 距离最大值，公式 (2.11)：

Δf = max_{D, D'} ‖ f(D) − f(D') ‖_1    (2.11)

然后给输出每个分量独立加一个拉普拉斯分布噪声，公式 (2.12)：

M(D) = f(D) + Lap(0, b)^d,    b = Δf / ε    (2.12)

Lap(0, b) 是均值 0、尺度参数 b 的拉普拉斯分布，密度函数 p(x) = (1/2b) exp(−|x|/b)，方差 2b²。这样得到的 M 严格满足 (ε, 0)-差分隐私。

本文怎么用？把客户端要上传的物品嵌入张量当作 f(D)，先做梯度裁剪把 L1 敏感度固定成 Δf = 1，再向嵌入张量的每个元素独立加 Lap(0, b) 噪声。单轮上传的隐私预算 ε₀ 与噪声尺度 b 的关系是 ε₀ = 1 / b，本文取 b = 0.01，对应 ε₀ = 100。

### 2.6.3 朴素组合、高级组合与 Rényi 差分隐私

差分隐私有个重要性质叫组合性。若 M₁ 满足 (ε₁, δ₁)-DP、M₂ 满足 (ε₂, δ₂)-DP，则二者的串行组合 M₁ ∘ M₂ 满足 (ε₁ + ε₂, δ₁ + δ₂)-DP。这是朴素组合定理。在本文场景下，每轮上传满足 (ε₀, 0)-DP，T 轮训练总预算见公式 (2.13)：

ε_naive = T · ε₀    (2.13)

朴素组合简单直观，但很保守。比如 ε₀ = 100、T = 25 时，朴素总预算高达 2500，几乎丧失了实际隐私意义。

Dwork 等人 2010 年证明了一个更紧的高级组合定理，公式 (2.14)：

ε_adv = √( 2 T · ln(1/δ') ) · ε₀ + T · ε₀ · ( e^{ε₀} − 1 )    (2.14)

δ' 是为了换更紧的 ε 而引入的小失败概率。这个上界在 ε 较小、T 较大时显著优于朴素组合。

Mironov 2017 年提出的 Rényi 差分隐私（RDP）做得更紧。RDP 用 Rényi 散度刻画机制的隐私性，机制 M 满足 (α, ρ)-RDP 当且仅当对任意相邻数据集 D, D' 满足公式 (2.15)：

D_α( M(D) ‖ M(D') ) ≤ ρ    (2.15)

D_α 是 α 阶 Rényi 散度。RDP 最大的好处是组合性极强：T 次同分布机制的组合预算严格等于 T · ρ，没有对数因子也没有额外缩放。最后再把 (α, T·ρ)-RDP 转换成 (ε, δ)-DP，转换公式见 (2.16)：

ε(α) = ρ · T + ln(1/δ) / (α − 1)    (2.16)

α 越大，单轮 Rényi 散度越大但转换到 (ε, δ)-DP 的对数项越小，二者存在最优权衡点。本文对 α 集合 {2, 4, 8, 16, 32, 64, 128} 做扫描，取使 ε(α) 最小的 α 作为最终上界。对 b 尺度的 Laplace 噪声其 α-Rényi 散度有闭式解。在 ε₀ = 100、T = 25、δ = 1e-5 的设置下，三种组合方法的对比见表 2.1。

表 2.1 三种隐私组合方法的预算对比（ε₀ = 100, T = 25, δ = 1e-5）

| 组合方法 | 总隐私预算 ε | 相对朴素紧度 |
| --- | --- | --- |
| 朴素组合 ε_naive = T · ε₀ | 2500 | 1× |
| Dwork 高级组合 | ≈ 1014 | ≈ 2.5× |
| Rényi DP (本文采用) | ≈ 300 | ≈ 8× |

Rényi DP 比朴素组合紧约 8 倍，比 Dwork 高级组合还紧约 3 倍，且不增加任何噪声本身——这是分析层面的免费收益。

## 2.7 评估指标

本文用 HR@10 与 NDCG@10 两个指标。设测试用户集合为 U_test，用户 u 的真实物品记为 i_u^*，模型为 u 输出的 Top-K 推荐列表记为 R_u^K。HR@K 与 NDCG@K 的算法见公式 (2.17) 与 (2.18)：

HR@K = (1 / |U_test|) ∑_{u ∈ U_test} 𝟙[ i_u^* ∈ R_u^K ]    (2.17)

NDCG@K = (1 / |U_test|) ∑_{u ∈ U_test} 𝟙[i_u^* ∈ R_u^K] · log 2 / log(1 + rank_u)    (2.18)

rank_u 是 i_u^* 在 R_u^K 中的位置（从 1 开始），𝟙[·] 是指示函数。两个指标的差别在于：HR@K 只看正样本是不是落进了前 K 名，对内部顺序无所谓；NDCG@K 通过对数折扣给排名靠前的命中更高奖励——排第 1 得 1 分，排第 2 得约 0.63 分，排第 10 得约 0.30 分。

实际意义上，HR 衡量的是召回能力（有没有把对的东西捞出来），NDCG 衡量的是排序质量（捞出来后排得好不好）。本文同时报这两个，方便从两个角度看模型表现。

## 2.8 工程工具

本文实现用 PyTorch 1.13 及以上版本，模型组织在 nn.Module 体系下，嵌入用 nn.Embedding，全连接层用 nn.Linear，可学习位置编码用 nn.Parameter，三个独立优化器都用 torch.optim.SGD。

数据预处理用 NumPy 与 Pandas，前者负责数值数组运算、后者负责处理交互记录的数据帧。留一法划分通过对每位用户的交互按时间排序后取倒数第一与倒数第二记录的方式高效实现。负采样在每一轮训练前重新采样以增加随机性。

scikit-learn 中的 pairwise_distances 函数用于计算用户两两之间的余弦距离，是构建双图的核心工具。需要特别注意：pairwise_distances 在 cosine 度量下返回的是距离 d ∈ [0, 2]——0 表示完全相同，越大越不相似——而不是相似度 s ∈ [−1, 1]。二者的关系是 s = 1 − d。这个接口约定容易踩坑，本文在调试中曾因此踩到一个 bug，详见 5.4 节。

训练过程的曲线可视化用 TensorBoardX，每轮记录 HR@10、NDCG@10、训练损失、假邻居率等关键指标。28 个单元测试用 pytest 框架组织，覆盖图构建对称性、对角线零值、HR/NDCG 公式正确性、可信邻居交集、并集、孤立节点回退、cosine 距离与相似度方向修复、多头注意力前向输出形状、Laplace 机制 ε 换算、Rényi DP 与朴素组合的紧度关系等关键方面。
